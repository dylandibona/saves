## 1. Executive summary

**The product works end-to-end for two users on `finds.dylandibona.com`.** The home page and AddForm are at the "premier" bar set by `STRATEGY.md` §"Brand" / `docs/design-direction.md`. Recent capture flow improvements are genuinely good — the live build animation, the Slackbot UA fix that unlocks Cloudflare-protected publishers, the Google Places API integration for Maps share URLs, the featured active-verb moment ("Reading…", "Distilling…") during enrichment.

**But the app is half-finished in two specific ways:**

1. **Six major surfaces are still on the pre-redesign aesthetic** — save detail, BuildPreview, map client, extracted section, settings, billing. The discontinuity between them and the new home/add surfaces is jarring on every tap.

2. **Media-type coverage is uneven.** Google Maps is excellent. Generic articles (including paywalled and Cloudflare-protected) are excellent. Instagram is mid. **YouTube is bare-minimum and hardcoded to category `noted`.** TikTok / Spotify / Apple Music / Letterboxd / Goodreads / Twitter / Substack / news have no special handling at all.

Per `PLAN.md`, the active priority (`now`) is **#1 Capture flow at full quality**. The last two design-heavy sessions effectively worked on #2 ("Library at full quality") which is `next`. Defensible drift — you can't test capture without a feed to capture into — but worth naming so the next conversation can reset.

## 4. Enrichment pipeline by platform

The architecture is in `lib/enrichment/enrich.ts` (~580 lines) with helper modules. The hot loop is `enrichUrl(rawUrl)` which:

1. Detects the URL type via `detectUrlType` from `lib/utils/url-detect.ts` (line 35) — returns `'google_maps' | 'instagram' | 'youtube' | 'generic'`
2. Picks a User-Agent via `pickUserAgent(url)` (`enrich.ts` line 118)
3. Routes to one of four branches: Google Maps (line 413), Instagram (line 490), YouTube (line 569), or generic (line 588 onward)

Per-branch detail:

### Google Maps — ✅ excellent

- Detection: `maps.google.com`, `maps.app.goo.gl`, `goo.gl`
- UA: Facebook bot (gets clean redirect from shortlinks where Chrome UA hits a consent interstitial)
- Resolution chain: shortlink → full `?q=NAME,ADDRESS` URL → `extractMapsPlaceName` parses the first comma-segment (`lib/utils/url-detect.ts` line 113)
- Places API call: `lookupPlace(name, coords)` in `lib/enrichment/places.ts` does `searchText` with 500m bias, resolves photo via `skipHttpRedirect=true` to get a clean `lh3.googleusercontent.com` CDN URL
- Output: title (Places displayName), subtitle (formattedAddress), imageUrl (Places photo), category (Places types → save_category via `typesToCategory` in `places.ts`), extracted{address, hours, phone, website, priceLevel}

**Gating:** `GOOGLE_PLACES_API_KEY` is in `.env.local` but **not in Vercel yet.** Until set in production+preview, Places lookups silently fall back to OG/Claude path (still much better than pre-FB-bot-fix, but not the full enrichment).

### Generic articles — ✅ excellent

- UA: Slackbot link-expander (`Slackbot-LinkExpanding 1.0`). Wins where Chrome UA, FB-bot, Twitterbot, Googlebot, Bingbot all hit Cloudflare challenges with no usable content. Tested 2026-05-12 against `foodandwine.com/scorpion-bowl-cocktail-recipe-11969017`: 7 ingredients, 3 HowToStep instructions, 30-min totalTime, 4 servings.
- OG fallback in `fetchAndParse` (line 134) tolerates non-2xx responses when body is >5KB (handles WAFs that 403 with the actual page intact).
- Claude classifier (`classifyWithClaude`, line 225) reads OG + JSON-LD + body slice, returns category + per-category structured extraction.

### Instagram — 🟡 mid

- UA: Facebook bot
- Catches OG image + sometimes a caption parsed from `Username on Instagram: "caption"` title format
- URL pattern detection for Reel / Post / Video / Story types
- **Misses:** comments (where the kettlebell-workout often lives), full captions on most posts, audio transcription
- The Apify subscription decision is the unblock here. ~$30/mo. Discussion deferred per `PLAN.md` 3d.

### YouTube — 🟥 bare minimum

`lib/enrichment/enrich.ts` line 569:

```ts
if (urlType === 'youtube') {
  const fetched = await fetchAndParse(rawUrl)
  const og = fetched?.og ?? { title: null, description: null, image: null, siteName: null }
  return {
    title: og.title,
    subtitle: og.siteName ?? null,
    category: 'noted',  // YouTube doesn't cleanly map — let user override
    imageUrl: og.image,
    canonicalUrl: rawUrl,
    coords: null,
    note: null,
    confidence: 'low',
    source: og.title ? 'og' : 'heuristic',
  }
}
```

**Hardcoded category `noted`**, no Claude pass, no channel/duration/view-count, no transcript. Every saved YouTube video — workouts, trailers, music videos, podcasts, video essays — gets routed to the lowest-signal category. **This is the most embarrassing miss in the current pipeline.**

Fix is cheap: use YouTube oEmbed (`https://www.youtube.com/oembed?url=...&format=json`, public, no key) for title + channel + thumbnail, then pass through `classifyWithClaude` so it can pick `movie` / `music` / `podcast` / `workout` / `article` based on title + channel + description.

### Not detected (fall through to generic)

| Platform | Behavior today | Risk |
|---|---|---|
| TikTok | Generic + Slackbot UA. Often returns no OG. | High — share-from-TikTok is very common, and what we return is usually nothing. |
| Spotify (track/album) | OG title present, but `extracted` lacks artist/album/duration | Medium — falls into `music` or `podcast` via Claude but no structured data. |
| Apple Music / Apple Podcasts | OG decent | Medium — similar to Spotify. |
| Letterboxd / IMDB | OG decent | Low — Claude usually catches year + director. |
| Goodreads / Amazon books | OG decent | Low — Claude usually catches author. |
| Twitter / X | Mostly locked down; OG often empty | High — common share source, low success rate. |
| Substack / Medium | Slackbot UA works | Low — generic path is fine here. |
| Reddit | OG usually fine | Low. |
| News paywalls (NYT/WSJ) | Slackbot UA usually 403s | Medium — Apple News / IG-shared NYT links land as link-only. |

## 6. Architectural assets accumulated since 2026-05-11

These exist and are worth knowing about in a fresh session — they're load-bearing for current behavior and changing them has downstream effects:

### `lib/enrichment/enrich.ts` (~580 lines)

- `pickUserAgent(url)` — per-host UA strategy. Today: FB-bot for Instagram + Google Maps shortlinks, Slackbot for everything else. **Touch carefully** — every WAF we've gotten through depends on this.
- `fetchAndParse(url)` — sends full browser headers + tolerates non-2xx with >5KB body. Returns `{ finalUrl, html, og }`.
- `classifyWithClaude(url, og, html, hint?)` — Claude opus-4-5 with a category-constraining hint (used by Maps branch to force `place|restaurant|hotel|event`).
- `enrichUrl(rawUrl)` — public entrypoint. Used by `/api/share-save` (iOS Shortcut path) via `lib/actions/enrich-url.ts` Server Action wrapper.

### `lib/enrichment/places.ts`

- `lookupPlace(query, coords)` — Places API (New) `searchText` with 500m bias.
- `typesToCategory(types, primary)` — maps Places types to our `save_category` enum.
- `resolvePhotoUrl(photoName, apiKey)` — uses `skipHttpRedirect=true` so the photo URL we store has no API key embedded.
- `summarizeHours()` — condenses Google's verbose `weekdayDescriptions` to one line.

### `app/api/enrich-stream/route.ts` (SSE)

Phased event stream consumed by `components/add/add-form.tsx`. Events:
`detected → fetching → og_parsed → place_looking_up → place_found → classifying → classified → titled → subtitled → noted → extracting_section → ingredient/instruction/exercise/place_detail/article_detail/movie_detail/product_detail/media_detail → coords → complete`

Pacing constants: `PACE.betweenPhases=90ms`, `PACE.betweenStructured=70ms`, `PACE.finalBeat=200ms`. Tune for animation feel.

### `components/add/build-preview.tsx`

`BuildState` shape + `BuildPreview` consumer + `ActiveVerb` + `TypingDots`. **On old design tokens** (24 old / 2 new). Slated for design sweep — see §8.

### Billing scaffold (Stripe, gate-open)

- `lib/billing/stripe.ts` — server-only client + PLANS map (free/personal/household_member)
- `lib/billing/can-save.ts` — `userCanSave(userId)` returns `{ok: true}` while `BILLING_ENFORCED !== 'true'`
- `/api/checkout` — initiates Checkout session
- `/api/stripe-webhook` — idempotent webhook handler with sig verification
- `/billing` — URL-only page (no nav link) for current plan + tier compare

**Gate is open.** One env var flip (`BILLING_ENFORCED=true`) activates enforcement across the chain. `PLAN.md`-aligned; not active by design.

### Security hardening migration (May 13)

`supabase/migrations/20260513000001_security_hardening.sql` — pinned `SET search_path = ''` on all our SECURITY DEFINER functions, REVOKE EXECUTE from anon where appropriate, COMMENT ON TABLE `stripe_events` documenting the intentional default-deny. Resolved 11 of 25 Supabase advisor lints. See `docs/session-notes-2026-05-13.md` for the architectural learning about RLS-pathway EXECUTE checks.

### Design system foundation

`app/globals.css`:
- Warm paper-ink dark palette (`--color-bg` etc. on `:root` via `@theme`)
- 14 category-color vars on bare `:root` (outside `@theme`) — **critical**: Tailwind v4 tree-shakes vars inside `@theme` that aren't used as utility classes; cat colors used only via `var()` in inline styles must live on raw `:root` or they vanish from the bundle. See commit `bc9fb62` and `docs/session-notes-2026-05-12.md` §"Architectural learnings."
- `.font-display`, `.font-serif`, `.font-serif-display`, `.font-semibold-ui`, `.font-body` classes wired to Bricolage Grotesque (variable, `wdth` + `opsz` axes) and Fraunces (`opsz=144`).

### Bottom nav + raised FAB

`components/bottom-nav.tsx` — top-left wordmark + 3 tabs (Library, Map, Settings) + centered raised FAB to `/add`. Hidden on `/login` and `/auth`. The masthead wordmark and the page hero italic numeral are visually unified (same Fraunces italic, two scales).

## 11. The honest one-paragraph summary

Finds works. The home page and AddForm look premier.
